# 🏗️ Padrões de Projeto Estruturais em Python

Este repositório contém uma pesquisa autoral e implementações práticas sobre **Padrões de Projeto Estruturais**, com o objetivo de entender como organizar classes e objetos para formar estruturas maiores e mais flexíveis.

---

## 📚 Padrões Pesquisados
A pesquisa teórica abrangeu os seguintes padrões:
* **Adapter:** Converte a interface de uma classe em outra esperada pelo cliente.
* **Bridge:** Separa a abstração da sua implementação.
* **Composite:** Compõe objetos em estruturas de árvore.
* **Decorator:** Adiciona responsabilidades a objetos dinamicamente.
* **Facade:** Fornece uma interface simplificada para um sistema complexo.
* **Flyweight:** Compartilha estados para suportar grandes quantidades de objetos.
* **Proxy:** Fornece um substituto ou marcador para controlar o acesso a outro objeto.

---

## 💻 Implementações Práticas

Foram escolhidos dois padrões para implementação em **Python**:

### 1. Adapter (`adapter`)
Resolve o problema de incompatibilidade entre uma classe de cálculo de área legada e uma nova interface de formas geométricas.

### 2. Decorator (`decorator`)
Demonstra como adicionar opcionais (leite, açúcar, chantilly) a uma bebida base sem criar uma explosão de subclasses via herança.
